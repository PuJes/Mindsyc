## 1. 产品概述 (Product Overview)

### 1.1 背景与痛点
当前知识管理工具市场呈两极化：
1.  [cite_start]**存储/收藏类（如 Cubox）**：解决了“稍后阅读”和信息聚合问题，但弱于深度结构化和知识内化 [cite: 42, 44, 55]。
2.  [cite_start]**表达/展示类（如 XMind/幕布）**：解决了思维可视化问题，但依赖人工手动录入和整理，维护成本高 [cite: 63, 65]。

**核心痛点**：
* [cite_start]**处理繁琐**：从视频/文章到“转文字→摘要→做脑图”的流程重复且耗时 [cite: 5]。
* [cite_start]**融合困难**：新观点与旧知识体系的对比、去重、整合完全依靠人脑记忆和比对，效率极低 [cite: 6]。
* [cite_start]**体系割裂**：信息来源太杂（B站、播客、研报），导致知识碎片化，难以形成统一的逻辑脉络 [cite: 4, 12]。

### 1.2 产品愿景
[cite_start]打造一款 **“主动演化”** 的知识体系工具。它不只是存储信息，而是像“外挂大脑”一样，自动“理解”外部信息（特别是视频/音频），并将其 **结构化地嫁接** 到用户既有的知识图谱中 [cite: 65]。

### 1.3 核心价值主张
* [cite_start]**从“被动记录”到“主动演化”**：自动对比新旧知识，提示融合建议 [cite: 65]。
* [cite_start]**视频即知识**：全链路自动化处理视频源（转录-提取-挂载） [cite: 58]。
* [cite_start]**可溯源的思考**：脑图节点直接链接到视频的时间戳或文章段落 [cite: 61]。

## 3. 功能需求详解 (Functional Requirements)

### 3.1 采集与预处理模块 (Ingestion & Processing)
**目标**：将多源异构信息转化为统一的“原子化”数据。

* **多源输入**：
    * 支持上传视频/音频文件、粘贴 URL（B站/YouTube/公众号/播客）。
    * [cite_start]支持浏览器插件采集网页正文 [cite: 28, 70]。
* **智能转录与解析**：
    * [cite_start]**音视频转录**：自动分离说话人，生成带时间戳的逐字稿 [cite: 21, 72]。
    * **术语词典匹配**：针对特定领域（如“固态电池”），识别专业术语，减少转录错误。
    * [cite_start]**人工校正模式**：对于置信度低的转录片段，提供高亮提示供用户快速修正 [cite: 39]。

### 3.2 认知理解模块 (Understanding & Atomization)
**目标**：从线性文本中提取逻辑结构。

* **智能分章**：基于语义密度和主题转折，自动将长视频/长文切分为章节。
* **要点卡片化 (Key Feature)**：
    * 从每个章节中提炼核心观点，生成 **“知识卡片”**。
    * [cite_start]卡片包含：标题、摘要、论据、**原文溯源锚点（时间戳）** [cite: 22, 95, 179]。
* **导图雏形生成**：
    * [cite_start]将零散卡片自动组织为树状结构（主题 → 子主题 → 论据），形成临时的“内容脑图” [cite: 23, 75]。

### 3.3 知识融合引擎 (Fusion Engine - **Core Differentiator**)
**目标**：将“新内容脑图”与“旧知识体系”进行对齐。

* **基底框架选择**：
    * 若用户无旧脑图：提供行业模板（投研/产品/学术）或 AI 根据当前内容自动生成基底框架。
    * 若用户有旧脑图：支持导入 XMind/OPML 作为参照系。
* **语义对齐与建议 (Semantic Matching)**：
    * [cite_start]系统自动比对新旧节点，提供三种操作建议 [cite: 24, 76]：
        1.  **合并 (Merge)**：新观点是旧节点的补充（如：将“QS电池技术”挂入“固态电池-技术路线”下）。
        2.  **新增 (Add)**：新观点填补了旧体系的空白（如：新增“AI Agent开发流程”分支）。
        3.  **忽略 (Ignore)**：内容重复或无关紧要。
* **冲突检测 (Conflict Detection)**：
    * 当新证据与旧结论矛盾时（例如：旧图“2025量产” vs 新视频“延期至2027”），触发 **冲突卡片**。
    * [cite_start]提供决策选项：保留旧观点 / 替换为新观点 / 并存（添加批注） [cite: 13, 77]。

### 3.4 交互与反馈模块 (Interaction & Output)
**目标**：用户决策与最终成果输出。

* **对齐确认面板**：
    * 左侧显示旧脑图结构，右侧显示待处理的新卡片。
    * [cite_start]中间区域显示 AI 的连线建议（合并/新增），用户可一键接受或拖拽修改 [cite: 16]。
* **增量更新记录**：
    * [cite_start]显示本次整合新增了多少节点、修改了多少结论，提供成就感反馈 [cite: 18]。
* **格式兼容与导出**：
    * 支持导出为 `.xmind`、`.opml`、Markdown 大纲。
    * [cite_start]**溯源链接保留**：导出后的文件，节点备注中应包含回跳链接（点击跳转回本工具播放对应视频片段） [cite: 25, 61]。

## 4. 非功能需求 (Non-functional Requirements)

* [cite_start]**准确性**：转录准确率需达到行业标准（如 >95%），摘要不出现严重幻觉（Hallucination） [cite: 35]。
* **性能**：1小时视频的处理（转录+提炼）应在用户可接受时间内完成（例如 <5分钟）。
* [cite_start]**隐私**：针对企业/投研用户，需确保上传的内部文档和私有脑图数据隔离，不用于训练公有模型 [cite: 40]。
* [cite_start]**兼容性**：导出的文件必须能被主流脑图软件（XMind, MindManager, 幕布）无缝打开 [cite: 38]。

## 5. 关键页面流 (Key User Flows)

[cite_start]基于提供的 Mermaid 流程图 [cite: 81, 114, 147]：

1.  **首页/工作台**：新建任务（导入链接/文件） -> 选择目标知识库（新建或选择已有）。
2.  **处理中状态**：显示进度条（转录中 -> 提取中 -> 构建中）。
3.  **结构化预览页**：左侧播放器/原文，右侧生成的“导图雏形”。用户可在此修正错误的摘要。
4.  **融合对齐页 (Killer Feature)**：
    * **差异视图 (Diff View)**：高亮显示“新知识”将如何改变“旧结构”。
    * **冲突弹窗**：当检测到矛盾时弹出，要求用户裁决。
5.  [cite_start]**结果页**：全屏展示更新后的最终脑图，支持节点展开/折叠，点击节点弹出“证据抽屉”回看视频 [cite: 80]。

## 6. 竞品差异化策略 (Competitive Strategy)

| 维度 | Cubox / Read-later Apps | XMind / MindMap Tools | **本工具 (KnowledgeFlow)** |
| :--- | :--- | :--- | :--- |
| **核心动作** | 收藏、高亮、全文检索 | 手动绘图、排版、展示 | **理解、语义匹配、自动嫁接** |
| **处理对象** | 网页/文章为主 | 用户脑中的想法 | **视频/音频** + 存量脑图 |
| **结果形态** | 列表/卡片流 | 静态图片/文件 | **动态生长的知识树** |
| **用户价值** | “我存下来了” | “我画出来了” | [cite_start]**“我学进去并融合了”** [cite: 65] |

## 7. 路线图规划 (Roadmap)

### Phase 1: MVP (最小可行性产品)
* 支持单视频导入 + 自动转录 + 摘要。
* 自动生成导图雏形。
* 支持导入 OPML 作为基底。
* 基础的“追加模式”（只能在末端新增，暂不支持复杂的冲突检测）。
* 导出为 Markdown/OPML。

### Phase 2: 智能增强 (Core Value)
* 实现节点级语义匹配（AI 判断 Merge/Add）。
* 开发“冲突检测”功能。
* 支持浏览器插件采集图文。
* 实现点击节点回看视频时间戳的溯源功能。

### Phase 3: 生态与协作
* API 接入（Notion, Obsidian 插件）。
* 支持多文件同时导入分析（例如：上传 5 份财报，生成一个综合脑图）。
* 团队协作版（多人维护同一个知识库）。
